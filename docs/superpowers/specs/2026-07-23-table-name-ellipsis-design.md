# 右侧数据表列表表名省略设计

## 背景与目标

右侧数据库概览中的数据表列表已对“注释”列启用文本省略，但“表名”列在名称较长时会撑开或挤压列表。目标是让超长表名与注释使用一致的单行省略表现，同时保留现有表类型图标、排序、点击打开表详情和列宽调整行为。

## 方案比较

1. 为“表名”列启用 Ant Design Table 的 `ellipsis` 配置（采用）。它与现有“注释”列保持一致，改动最小，继续遵循列表组件对列宽和虚拟滚动的既有处理。
2. 只在表名文本节点上增加 CSS 截断。需要额外处理含图标的 flex 容器宽度，且容易与列宽调整机制脱节。
3. 手动截取表名字符串。会丢失完整名称，且不能随列宽变化自适应，因此不采用。

## 实现设计

- 在 `src/components/database/DatabaseOverview.tsx` 的 `name` 列定义中增加 `ellipsis: true`。
- 不修改默认列宽、列顺序、表名渲染内容或鼠标点击逻辑。
- 不增加新状态、样式文件或后端改动。

## 验证

- 在 `src/__tests__/DatabaseOverview.test.tsx` 增加组件测试，断言传给数据表列表的“表名”列表定义启用了省略配置。
- 执行该测试文件，并运行前端构建以验证 TypeScript 与生产构建。

## 范围外

- 不调整表名列默认宽度或用户已保存的列宽。
- 不改变工具提示、排序或长表名的原始数据。
