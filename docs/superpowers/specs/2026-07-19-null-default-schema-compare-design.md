# 数据结构对比 NULL 默认值等价设计

## 背景

数据结构同步完成后，源端与目标端字段的默认值可能分别由数据库元数据表示为“无默认值”和显式 `DEFAULT NULL`。前端会把无默认值显示为 `NULL`，因此用户看到两端均为 `NULL`，后端却因 `None` 与 `Some("NULL")` 原始值不同而报告“结构变化”。

当字段的其他结构属性一致时，这属于假差异，不应继续出现在对比结果或同步计划中。

## 目标

- 将无默认值与语义上的空默认值视为等价。
- 支持大小写不敏感且可由完整外层括号包裹的 `NULL`，例如 `NULL`、`null`、`(NULL)` 和 `(( NULL ))`。
- 保留源端和目标端的原始快照值，不改变展示或后续 DDL 构建所使用的数据。
- 不把字符串字面量 `'NULL'` 或其他真实默认表达式误判为空默认值。

## 方案

在后端结构对比模块增加默认值语义比较辅助函数，并由字段差异判定逻辑调用。

比较规则如下：

1. `None` 视为语义空默认值。
2. 字符串先去除首尾空白；若整个表达式被一对匹配的外层括号包裹，则逐层去除外层括号与其内部首尾空白。
3. 规范化后的裸标识仅在大小写不敏感等于 `NULL` 时视为语义空默认值。
4. 两侧均为语义空默认值时判定相等。
5. 其他情况继续比较未经改写的原始默认值，避免扩大现有默认表达式归一化范围。

该逻辑只改变 `default_value` 是否进入 `changed_fields`，不修改快照采集、序列化、前端格式化或数据库同步方言。

## 边界

- `'NULL'` 是字符串默认值，不与无默认值等价。
- `NULL::text`、`CAST(NULL AS ...)` 等表达式不在本次等价范围内。
- 只有完整包裹表达式的平衡外层括号才会被剥离；包含其他内容或括号不平衡的表达式保持原样。
- 非空默认表达式继续保持大小写、空白和括号敏感的现有比较行为。

## 测试

在结构对比模块的单元测试中覆盖：

- 源端无默认值、目标端 `NULL` 时不产生表或字段差异。
- 源端 `(( NULL ))`、目标端无默认值时不产生差异。
- 两端分别为不同大小写或不同完整外层括号形式的空默认值时不产生差异。
- 无默认值与 `'NULL'`、`0` 等真实默认值仍报告 `default_value` 变化。
- 现有字段属性差异与稳定排序测试继续通过。

## 验收标准

成功同步后，如果字段其他属性一致且两端默认值均语义等价于 `NULL`，结构对比结果不再显示“结构变化”，同步计划也不再生成对应默认值操作；真实默认值差异仍被保留。
