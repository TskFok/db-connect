# 数据库结构同步功能设计

## 背景与目标

DB Connect 已支持在两个不同的同类型保存连接之间比较数据库/schema，比较范围为物理表和字段结构。现有对比只读，不生成或执行 DDL。

本功能在一次成功的数据库对比之后提供结构同步入口。用户可以按表选择部分差异，或选择全部差异表；系统把源端的物理表与字段结构同步到目标端。所有写入必须先生成并展示后端计划和 SQL，用户确认后才执行。

同步必须满足以下安全目标：

- 不信任或直接执行前端提供的 SQL。
- 执行前重新校验两端结构，拒绝过期计划。
- 删除操作默认关闭，并经过额外确认。
- 不承诺五类数据库无法共同保证的整批事务回滚。
- 不影响当前工作区连接，也不泄露保存连接凭据。
- 元数据继续按数据库/schema 批量查询，不在表循环中查询 SQL。

## 范围

### 包含

- MySQL/MariaDB、PostgreSQL、SQLite、SQL Server、ClickHouse 全部现有数据库类型。
- 仅在两个不同的同类型保存连接之间同步。
- 复用现有源端、目标端数据库/schema 选择。
- 按表选择部分差异表。
- 选择全部符合条件的差异表进行同步。
- 同步物理表和字段结构：
  - 创建源端独有表。
  - 新增源端独有字段。
  - 修改同名字段的类型、可空、默认值、主键、额外属性、注释与字段顺序；实际执行能力由方言安全边界决定。
  - 在显式开启删除后删除目标端独有字段或表。
- 执行前预览表级操作、风险等级和完整 SQL。
- 同步计划指纹与执行前结构漂移校验。
- 首个失败即停止，并返回已成功、失败和未执行操作。
- 全部成功后重新对比并刷新结果。

### 不包含

- 不同步表数据。
- 不比较或同步索引、外键、视图、触发器、例程、事件、权限及其他数据库对象。
- 不支持不同数据库类型之间转换。
- 不保存同步历史或可恢复快照。
- 不自动回滚整批 DDL。
- 不用重建整表强行模拟数据库原生无法安全表达的变化。
- 不允许前端编辑后端生成的 SQL 后再执行。

## 已确认的产品决策

- 选择粒度为表级，不下钻到逐字段选择。
- 删除操作由「包含删除操作」开关控制，默认关闭；开启后展示危险项并二次确认。
- 首期覆盖全部五类数据库，但方言无法安全表达的变化显示为阻塞项，不生成近似 SQL。
- 首期只负责物理表与字段，其他对象后续扩展。
- 批量执行遇到首个失败立即停止，保留此前已成功的变更。
- 所有同步始终先预览操作和 SQL，再由用户确认执行。
- 后端采用无状态的两阶段计划，并在执行前重新采集结构、校验计划指纹。

## 用户流程与界面

### 差异选择

现有数据库对比成功且存在差异时，结果表增加表级复选框。原有搜索、状态筛选和字段展开继续保留。

选择规则：

- 表名是选择的唯一稳定标识。
- 表头全选覆盖全部符合条件的差异表，不受当前搜索和状态筛选影响。
- 搜索或切换状态筛选不会清除已有选择。
- 关闭删除开关时，纯目标端表不可选。
- 用户关闭删除开关时，已选择的纯目标端表自动取消选择。
- 关闭删除开关时，变化表仍可选择，但其中目标端独有字段的删除操作会被跳过，并在预览中明确显示。
- 对比端点变化、交换两端、重新对比或关闭窗口时清空选择和旧计划。

结果区工具栏增加「包含删除操作」开关。弹窗底部在存在差异时增加「预览同步（N）」主操作；没有选中表、正在对比、导出、预览或执行时按状态禁用。

### 同步预览

点击「预览同步」打开独立预览窗口。窗口显示：

- 明确的源端到目标端方向，以及两端连接和数据库/schema 名称。
- 选中表数、可执行操作数、高风险操作数、删除操作数和阻塞项数。
- 按执行顺序排列的表级操作。
- 每个操作的说明、风险等级和完整 SQL。
- 关闭删除时被跳过的删除差异。
- 方言无法安全表达的阻塞原因。
- “DDL 不保证整批回滚，失败时停在首个失败项”的固定提示。
- 计划指纹的短格式，便于识别预览是否更新。

计划存在阻塞项时禁用确认执行。用户可以返回对比结果取消选择被阻塞的表，再重新生成预览。

没有阻塞项时，用户必须勾选「我已检查以上 SQL，并理解已成功执行的 DDL 可能无法自动回滚」。计划包含删除操作时，最终按钮使用危险样式并明确标注删除风险。

### 执行结果

执行期间锁定端点选择、关闭、交换、导出、预览和重复提交操作。首期不提供取消执行，因为部分数据库 DDL 无法可靠取消或回滚。

全部成功后：

- 展示已成功操作数量。
- 后端重新对比两端并返回最新结果。
- 前端以最新结果替换旧结果，清空选择和预览计划。
- 删除开关关闭时，目标端多余表/字段仍保留并继续显示差异。
- 已完全同步的表从差异列表中消失。

失败后：

- 展示已成功操作、首个失败操作及尚未执行操作。
- 若一个多语句操作中途失败，已执行语句仍明确计入成功记录。
- 不允许直接重试旧计划。
- 提供「重新对比」操作，让用户以目标端真实状态重新开始。

## 架构

采用“后端生成计划、前端只选择和确认”的无状态两阶段架构。

### 后端组件

- `commands/database_sync.rs`
  - 解析保存连接。
  - 校验端点和目标端写入资格。
  - 管理两侧临时连接生命周期。
  - 暴露预览与执行 Tauri 命令。
- `db/schema_sync/mod.rs`
  - 规范化选择。
  - 从两侧完整快照生成稳定、有序的同步计划。
  - 计算风险、跳过项、阻塞项和计划指纹。
  - 调度方言规划器和执行器。
- `db/schema_sync/mysql.rs`
- `db/schema_sync/postgres.rs`
- `db/schema_sync/sqlite.rs`
- `db/schema_sync/sqlserver.rs`
- `db/schema_sync/clickhouse.rs`
  - 负责各数据库 DDL 构造、能力判断、标识符转义和执行。
- 现有 `db/schema_compare`
  - 继续负责批量快照与差异算法。
  - 扩展内部快照以携带同步建表所需的结构化原生元数据。
  - 对外的现有对比契约保持兼容。

现有 `database_compare.rs` 中的临时连接、保存连接查找、资源清理和错误脱敏能力应抽到小范围共享模块，避免同步命令复制安全逻辑。

### 前端组件

- `DatabaseCompareModal`
  - 继续负责端点和顶层请求状态。
  - 持有当前对比结果与同步窗口开关。
- `DatabaseCompareResults`
  - 承担摘要、筛选、表格展开、表级选择和删除开关。
- `DatabaseSyncPreviewModal`
  - 展示计划、SQL、风险确认和执行结果。
- `services/tauriCommands.ts`
  - 增加预览和执行命令封装。
- `types/index.ts`
  - 增加同步请求、计划和执行结果契约。
- `utils/databaseSync.ts`
  - 只包含选择、标签和展示用纯函数，不生成 SQL 或重新计算后端计划。

拆分结果区和预览窗口可以避免继续扩大现有对比弹窗单文件，同时保持每个组件职责清晰。

## 两阶段命令

### `preview_database_sync`

输入：

```text
DatabaseSyncRequest
  source: DatabaseCompareEndpointRequest
  target: DatabaseCompareEndpointRequest
  selected_tables: string[]
  include_drops: boolean
```

行为：

1. 加载保存连接，校验两个连接不同且数据库类型一致。
2. 拒绝配置为只读的目标保存连接。
3. 对表名排序去重；拒绝空选择和空表名。
4. 并行建立两侧临时连接。
5. 校验数据库/schema 仍存在。
6. 分别批量读取两侧完整结构快照。
7. 重新计算差异，校验每个选择仍是差异表。
8. 由对应方言规划器生成操作、跳过项和阻塞项。
9. 对规范化请求、两侧相关快照和有序计划做规范序列化，并计算 SHA-256 计划指纹。
10. 清理两侧临时连接，返回预览。

该命令只读，不在目标端执行 DDL。

### `execute_database_sync`

输入：

```text
ExecuteDatabaseSyncRequest
  request: DatabaseSyncRequest
  plan_fingerprint: string
```

行为：

1. 重复预览阶段的所有端点、选择和只读校验。
2. 重新建立两侧临时连接并批量读取完整快照。
3. 使用同一纯规划器重新生成计划和指纹。
4. 指纹不一致时立即停止，执行零条 DDL，并返回“结构已变化，请重新对比”。
5. 计划包含阻塞项、没有可执行操作或删除保护不满足时执行零条 DDL。
6. 指纹一致时，在目标端临时写连接上按计划顺序执行。
7. 首条失败语句出现后停止，不执行剩余操作。
8. 全部成功时重新批量读取两端结构并计算最新对比结果。
9. 清理临时连接，返回结构化执行结果。

执行前的第二次读取是乐观并发校验。校验完成后若数据库被外部进程立即修改，无法形成跨产品的全局锁；后续 DDL 错误按首个失败停止规则处理。

计划操作 ID 由表名、操作类型和稳定序号生成，只用于关联预览和执行结果。SQL 不从前端回传，也不是执行请求的信任来源。

计划指纹不包含密码或其他凭据。选择表名先排序去重；快照字段和操作顺序均稳定化，保证相同结构产生相同指纹。

## 快照扩展

现有公开 `ColumnSnapshot` 足以展示差异，但同步不能只依赖 `extra` 等展示字符串。内部快照增加结构化的建表和列生成元数据，并继续由单次或固定次数的数据库/schema 级批量查询填充。

典型扩展包括：

- MySQL/MariaDB：建表所需引擎和表注释；字段生成表达式等原生信息。
- PostgreSQL：普通表与分区表标识、identity/generated 结构化信息。
- SQLite：原始声明及 generated/自增识别所需信息。
- SQL Server：identity、computed 表达式、用户定义类型信息。
- ClickHouse：engine、sorting key、partition key、primary key 表达式和字段 default kind/expression。

扩展元数据只用于正确创建表和判断规划能力，不扩大首期对比对象范围。某类特殊表无法在不带入索引、外键或其他排除对象的情况下完整表达时，将该表标为阻塞项。

## 计划生成与执行顺序

总体顺序为：

1. 创建源端独有表。
2. 新增字段。
3. 修改字段属性、注释和主键。
4. 删除目标端独有字段。
5. 删除目标端独有表。

表名和同阶段操作按稳定名称排序。方言规划器可以在单表内部调整依赖，例如先删除旧主键再建立新主键。所有显式删除操作始终排在非删除操作之后。

每个操作可以包含多条 SQL。执行结果按语句记录，确保多语句操作中途失败时不会错误宣称整个操作未发生。

## 删除保护与风险分类

### 删除保护

- `include_drops = false` 时不生成任何 `DROP COLUMN` 或 `DROP TABLE` 可执行 SQL。
- 纯目标端表记录为跳过项，且前端不可选择。
- 变化表中的目标端独有字段记录为跳过项；同表其他非删除变化仍可生成计划。
- `include_drops = true` 时才生成删除操作，并要求最终危险确认。

### 风险分类

- `normal`
  - 创建新的空表。
  - 新增不会删除现有数据的普通字段。
  - 更新注释。
- `high`
  - 修改已有字段的类型、可空、默认值、主键、extra 或顺序。
  - 新增非空字段、生成字段或可能扫描/重写目标表的操作。
- `destructive`
  - 删除字段。
  - 删除表。

类型是否“扩大”在五类数据库中缺少统一、可靠的判断，因此已有字段类型变化统一按高风险显示，不给出可能误导的“安全扩容”承诺。

## 方言规则

### MySQL/MariaDB

- 使用数据库限定名和反引号安全转义。
- 支持普通表创建、字段增删改、字段顺序、注释、主键和常见额外属性。
- 建表时使用批量快照取得的源端引擎等必要定义。
- 无法安全映射的生成表达式或特殊表属性标为阻塞项。

### PostgreSQL

- 使用 schema 限定名和双引号安全转义。
- 支持普通表创建、字段增删、类型、可空、默认值、注释和主键变化。
- PostgreSQL 不支持普通字段的物理顺序调整；仅字段顺序差异标为阻塞项。
- identity/generated 转换或分区表创建在无法由结构化快照完整表达时标为阻塞项。

### SQLite

- 使用 schema 限定名和双引号安全转义。
- 支持可由当前 bundled SQLite 原生表达的建表、新增字段和删除字段操作。
- 修改已有字段类型、可空、默认值、主键、生成属性或字段顺序通常需要重建表；首期不通过重建表兜底，因此标为阻塞项。
- 不因同步字段而丢弃目标端现有索引、外键或触发器。

### SQL Server

- 使用 schema 限定名和方括号安全转义。
- 支持普通表创建、字段增删、普通类型/可空/默认值/注释和主键变化。
- 字段物理顺序、identity/computed 转换及目标端缺失的用户定义类型标为阻塞项。
- 删除字段前的默认约束处理沿用现有安全 DDL builder。

### ClickHouse

- 使用数据库限定名和反引号安全转义。
- 建表时使用源端 engine、sorting key、partition key 和 primary key 表达式生成有效表定义。
- 支持数据库原生可表达的字段新增、修改、删除及 default/materialized/alias 定义。
- 无法通过安全 `ALTER` 表达的主键、排序键或引擎变化标为阻塞项；首期不重建表。

## 阻塞项规则

方言不能安全原地表达某项变化时，规划器返回阻塞项而不是近似 SQL。只要选中表中存在阻塞项，整个预览的 `can_execute` 为 `false`。

用户必须返回差异列表并取消选择对应表，才能执行其他表。系统不提供“忽略阻塞项并继续同步全部”的隐式行为，避免“同步全部”名义下产生不可见遗漏。

## 错误处理

预览和执行命令都直接拒绝以下无效请求：

- 保存连接不存在或已删除。
- 选择同一个保存连接作为两端。
- 两侧数据库类型不一致。
- 数据库/schema 不存在。
- 目标保存连接配置为只读。
- 没有选中表、表名为空或选择不再是差异表。

预览发现删除保护关闭、方言无法安全表达或没有可执行操作时，仍返回结构化的跳过项、阻塞项和 `can_execute = false`，让用户能看到原因；预览命令本身不因这些计划结果丢失说明信息。

执行命令重新生成计划后，若存在阻塞项、没有可执行操作，或计划尝试在 `include_drops = false` 时执行删除 SQL，则执行零条 DDL 并返回明确错误。

连接、权限、元数据和 DDL 错误须包含侧别、连接名称、数据库/schema、表名和操作说明，但不得包含密码或其他凭据。

计划指纹不一致时返回专用的结构漂移错误，且不得执行任何 DDL。

执行错误返回结构化部分结果。命令不得因为返回 `Err` 而丢失已经成功执行的操作记录。临时连接清理错误追加到 `cleanup_errors`；若没有执行任何 DDL，也可以作为普通命令错误返回。全部 DDL 成功但清理失败时，界面显示“同步完成，但连接清理异常”的警告，不把已落库操作描述为失败或回滚。

## 临时连接与信任边界

- 预览和执行都从加密保存连接中加载完整配置。
- 临时连接不注册到 `ConnectionManager`，不改变当前工作区连接。
- 网络 I/O 不在全局连接管理锁内执行。
- 源端仅执行元数据读取。
- 目标端只有 `execute_database_sync` 在指纹一致后执行 DDL。
- 前端只提交端点、表名、删除开关和计划指纹。
- 前端不提交 SQL，后端不执行前端提供的 SQL。
- 解密后的密码只存在于 Rust 内存，不进入计划、指纹、日志或返回值。

## 测试策略

### Rust 纯计划测试

- 源端独有表生成建表操作。
- 目标端独有表在删除关闭时跳过、删除开启时生成危险操作。
- 源端独有字段、新旧字段属性变化和目标端独有字段。
- 删除保护不产生任何删除 SQL。
- 普通、高风险、删除风险分类。
- 稳定表名、操作和语句排序。
- 选择排序去重和空选择校验。
- 相同结构产生相同指纹，任一相关结构变化导致指纹变化。
- 阻塞项使计划不可执行。

### 方言测试

五个方言分别覆盖：

- 数据库/schema、表名和字段名特殊字符转义。
- 创建普通表。
- 新增、修改和删除字段。
- 默认值、可空、注释、主键与 extra。
- 字段顺序规则。
- 多语句操作顺序。
- 各方言不支持变化准确生成阻塞项。
- SQL 不包含源端限定名误写到目标端。

### 命令与执行测试

- 保存连接、类型、数据库/schema、只读和删除保护校验。
- 预览阶段执行零条 DDL。
- 指纹一致才进入执行。
- 结构漂移时执行零条 DDL。
- 首个失败停止。
- 首条失败与部分成功两种结构化结果。
- 全部成功返回最新对比结果。
- 两侧正常、单侧失败、查询失败、执行失败和清理失败的资源释放。
- 数据库密码、SSH 密码和证书密码脱敏。

### 元数据与集成测试

- 五类数据库同步快照保持数据库/schema 级批量查询，不出现逐表查询。
- SQLite 临时数据库覆盖建表、新增字段、删除开关、结构漂移和同步后重新对比。

## 验收标准

- 五类现有数据库都显示结构同步入口，并能生成其安全可表达操作的预览和执行计划。
- 用户可按表选择部分差异，也可选择全部符合条件的差异表。
- 所有写入前都展示后端生成的有序 SQL 和风险等级。
- 删除默认关闭，只有显式开启并再次确认后才执行删字段或删表。
- 前端不能提供或篡改实际执行 SQL。
- 确认执行前结构发生变化时执行零条 DDL，并提示重新对比。
- 方言无法安全表达的操作明确阻塞，不静默忽略或近似执行。
- 执行遇到首个失败即停止，并准确报告已成功、失败和未执行操作。
- 全部成功后自动刷新对比结果。
- 同步不影响当前工作区连接，错误不泄露凭据。
- 同步元数据按端批量查询，不在表循环中查询 SQL。
- 全部自动化测试、构建、lint 和格式检查通过。
