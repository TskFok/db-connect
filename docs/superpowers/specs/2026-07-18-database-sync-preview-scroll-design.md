# 同步 SQL 预览滚动修复设计

## 背景

「同步 SQL 预览」弹窗在操作或 SQL 较多时，内容会超过可视高度，但中间内容区域无法上下滚动，导致底部内容不可访问。

修复后的交互应保持弹窗标题和底部操作按钮可见，仅让中间内容区域纵向滚动。同步预览态与执行结果态共用同一套滚动布局。

## 根因

当前弹窗把 `overflow-y: auto` 放在 `.database-sync-preview-body`，并依赖 `height: 100%` 获得滚动高度；它的父级 `.ant-modal-body` 只参与 flex 高度分配，没有确定高度，并设置了 `overflow: hidden`。

当内容超过视口时，内层百分比高度不能稳定形成受限滚动区域，父级又会裁切溢出内容，因此用户无法访问下方 SQL。

## 方案选择

采用项目内已有的 Ant Design Modal 滚动模式：

- Modal content 使用纵向 flex 布局并限制为当前视口可用高度。
- Modal body 使用 `flex: 1 1 auto`、`min-height: 0` 和 `overflow-y: auto`，成为唯一的纵向滚动容器。
- `.database-sync-preview-body` 只负责网格排版、间距和内边距，不再设置百分比高度或纵向滚动。
- 保留 SQL 代码块自身的横向滚动，长 SQL 不改变弹窗整体宽度。

不采用以下方案：

- 只在外部 CSS 中覆盖 Ant Design 内部类：改动虽小，但布局契约不易被组件测试直接保护。
- 固定整个弹窗高度并继续滚动内层：短内容也会被迫占满窗口，且需要重复计算标题和页脚高度。

## 组件与行为

`DatabaseSyncPreviewModal` 通过 Modal 的 `styles` 属性声明 content 与 body 的结构性布局。标题和 footer 位于可滚动 body 之外，因此滚动长内容时仍保持可见。

同步方向栏继续使用 `position: sticky`，在中间内容滚动时停留在滚动区域顶部。预览摘要、警告、阻塞项、跳过项、SQL 操作列表、确认区和执行结果按原顺序随 body 滚动。

本修复不修改：

- 同步计划、SQL 或执行结果的数据结构；
- 确认、执行、返回和重新对比逻辑；
- 执行锁、关闭限制与无障碍文案；
- 后端命令及数据库访问。

## 错误处理

本修复不引入新的运行时错误分支。预览为空、计划不可执行、存在阻塞项、执行成功或执行失败时，均复用同一个可滚动 Modal body。

## 测试策略

先增加会失败的组件回归测试，直接检查渲染后的 Ant Design Modal：

- content 是纵向 flex 容器，并受视口高度限制；
- body 可以收缩且设置 `overflow-y: auto`；
- 内层 `.database-sync-preview-body` 不再承担纵向滚动职责；
- 标题与 footer 位于 body 之外，原有预览和执行结果交互测试继续通过。

测试按以下顺序执行：

1. 运行 `DatabaseSyncPreviewModal` 聚焦测试，确认新增断言在修复前按预期失败。
2. 实施最小布局修改，确认聚焦测试通过。
3. 运行数据库同步相关前端测试、前端全量测试、类型构建和 lint。

## 验收标准

- SQL 或执行结果超过可视高度时，中间内容可通过鼠标滚轮、触控板和键盘上下滚动。
- 弹窗标题、关闭入口和底部操作按钮保持可见。
- 短内容弹窗不被强制撑满整个视口。
- 长 SQL 仍在代码块内横向滚动，不造成页面横向溢出。
- 预览确认、执行锁、结果展示和重新对比行为无回归。
