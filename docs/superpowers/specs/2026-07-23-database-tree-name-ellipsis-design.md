# 左侧数据库名称省略设计

## 背景与目标

左侧数据库树中的表名已支持单行省略和完整名称提示，但数据库名称仍以普通加粗文本渲染。过长数据库名称会占用树节点可用宽度。目标是让数据库名称与表名节点保持一致：在可用宽度不足时单行省略，同时保留完整名称的提示。

## 方案比较

1. 复用表名节点的 `Typography.Text` 省略配置（采用）。在数据库标题上增加相同的块级、最小宽度和单行截断样式，并使用 `ellipsis={{ tooltip: db }}` 展示完整名称；与现有树节点的行为一致。
2. 只添加 CSS `text-overflow`。实现较短，但不会提供完整名称提示，且会分散与表名节点相同的样式约定。
3. 截断数据库名称字符串。会永久丢失可见信息，也不能随侧边栏宽度调整，因此不采用。

## 实现设计

- 在 `src/components/database/DatabaseTree.tsx` 的数据库节点 `title` 中，为加粗 `Text` 添加与表名节点一致的单行截断样式和 `ellipsis={{ tooltip: db }}`。
- 不改变数据库节点的 key、图标、子表加载、选择、展开或右键菜单逻辑。
- 不修改侧边栏宽度、状态管理、样式文件或后端代码。

## 验证

- 在 `src/__tests__/DatabaseTreeCapabilities.test.tsx` 中增加长数据库名称的组件测试，断言对应数据库节点启用了 Ant Design 的省略类。
- 执行该测试文件及前端生产构建。

## 范围外

- 不改变连接名称、多连接标签或右侧内容区标题的显示行为。
- 不调整数据库名称排序和原始名称数据。
